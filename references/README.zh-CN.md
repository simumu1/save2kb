# web-to-kb 技能说明书（中文）

## 简介

**web-to-kb**（网页→知识库）是一个通用技能：把任意网页文章读取、转成 Markdown、存到结构化知识库，并自动分类。

**✅ 轻松读取各网站文章 — 公众号/知乎/头条/博客/新闻，你只管发链接。**

## 适用场景

- 用户发来公众号/知乎/头条/博客/新闻文章链接，说"存知识库"
- 定期搜集行业文章，归档到知识库
- 知识库周报/系列总结

## 智能体使用流程

### 触发条件

用户发链接 + 提及"知识库"关键词（如"存知识库"、"收入知识库"等）。

### 执行步骤

```
用户发链接 + "知识库"
    ↓
① 读取文章：web_extract → Camofox（自动启动）→ browser_navigate
    ↓
② 汇报分析结果（要点、核心观点）
    ↓
③ 询问知识库："要存知识库吗？有现成的还是帮你建一个？"
    ├─ "有（Obsidian/特定路径）" → 确认路径 → 自动分类存档
    ├─ "没有" → "要帮你建吗？（~/知识库/）"
    │           ├─ "好" → 创建 → 存档
    │           └─ "不用" → 结束
    └─ 之前确认过 → 直接存档
```

### 读取策略（省 token）

| 优先级 | 方法 | 说明 |
|:------:|:----|:-----|
| 1 | web_extract | 最快，设 char_limit=10000。⚠️ 部分环境 backend 为搜索only，可能失败 |
| 2 | browser_navigate | 通过 Camofox 反检测浏览器。先检查 localhost:9377，未运行则自动启动 |
| 3 | web_search + 转载 | 前两者都失败时，搜转载内容 |

### 知识库结构

```
~/知识库/
├── INDEX.md              ← 主索引（按时间倒排）
├── 原始文章/              ← 文章原文MD，YYYY-MM-DD-标题.md
├── 主题归类/              ← 按主题分类的提炼笔记
└── 系列总结/              ← 周期周报/汇总
```

### 分类规则

- 根据文章内容自动判断标签（A股/量化/AI/宏观/行业/科技/财经/SEO/其他）
- 除非用户主动指定类别，否则不问用户
- 内容跨主题时选最核心的1-2个标签
- 可新建合适标签

### 不存知识库的情况

用户只发链接（没提"知识库"）→ 只读+分析汇报，不存档不询问。

## 环境依赖

- **Camofox**：local 模式，`~/camofox-tmp/`，npm start 启动于 localhost:9377
- **web_extract**：内置 Hermes 工具，需配置非 DDG backend 才能直读

## 注意事项

- **纯文字存档**：知识库只存文章的文字/纯文本内容，不包含图片、视频、排版样式。原文链接保留在 frontmatter 中，需要看图时点开原链接即可
- 同一 URL 不重复保存（检查 INDEX.md）
- 文件名截取标题前 30 字
- Camofox 启动需 ~8 秒，若持续运行则后续快速
